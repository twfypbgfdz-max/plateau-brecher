# Projektregeln

## Grundsatz

Erst analysieren, dann kurz Patchplan nennen, dann gezielt aendern.

## Arbeitsbereich

- Ausschliesslich in `07_KALORIEN_APP/` arbeiten.
- Keine bestehenden Projektdateien ausserhalb dieses Ordners aendern.
- Keine Plateau-Brecher-Dateien anfassen.
- Keine Stable- oder Live-Dateien ueberschreiben.
- Keine Dateien loeschen.
- Keine Deployments.
- Keine GitHub-Pushes.

## Produktregeln

- Eigenstaendige Kalorien- und Ernaehrungs-App.
- Mobile First.
- MVP zuerst.
- Keine KI im MVP.
- Keine grosse Lebensmitteldatenbank im MVP.
- Keine unnoetigen Refactors.
- Kleine, nachvollziehbare und rueckbaubare Patches.

## Nach jedem Patch

Kurz berichten:

- welche Dateien geaendert wurden
- welche Version betroffen ist
- welche Tests gemacht wurden
- was manuell getestet werden soll
